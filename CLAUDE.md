# CLAUDE PROJECT SKILL

You are working on AIO Digital Solutions Website.

Always follow these rules.

## Architecture Rules

* Follow Clean Architecture strictly.
* No business logic inside components.
* Use Domain -> UseCase -> Repository pattern.
* Use TypeScript everywhere.
* Avoid any.

## UI Rules

* Use Shadcn UI.
* Use TailwindCSS.
* Use Framer Motion.

Every page must:

* Desktop optimized
* Tablet optimized
* Mobile optimized

## Design Rules

The website must look:

* Enterprise
* Futuristic
* Premium
* Technology-focused

Avoid:

* Generic ecommerce design
* Basic bootstrap layout
* Old-fashioned UI

## Component Rules

Create reusable components.

Examples:

components/common

* Button
* Card
* Modal
* SectionHeader
* StatCard

components/business

* ProductCard
* SolutionCard
* ProjectCard
* NewsCard
* BrandCard

## Performance Rules

Always use:

* Lazy Loading
* Memoization
* Dynamic Imports

## Image Rules

Images come from:

public/images

Never use external image URLs.

## Admin Rules

All CRUD modules must support:

* Create
* Read
* Update
* Delete

Use:

* TanStack Table
* Search
* Filter
* Pagination

## Data Rules

Create realistic mock data.

Products:

Minimum 50 products.

Projects:

Minimum 30 projects.

News:

Minimum 50 articles.

Brands:

Minimum 10 brands.

## UX Rules

Every page should include:

* CTA section
* Related content section
* Contact CTA

## SEO Rules

Every page must have:

* Meta Title
* Meta Description
* OpenGraph
* Structured Data

## Coding Rules

* Feature-based architecture
* Atomic reusable components
* Strong typing
* Clean naming convention

## Output Rules

Generate complete files.

Never generate placeholders.

Never generate pseudo code.

Always generate production-ready code.
